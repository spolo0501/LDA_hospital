# Claude Code 自定義命令使用指南

本目錄包含了專案的自定義 Claude Code 命令，用於快速引導 Claude 遵循專案的目錄架構規範。

---

## 可用命令列表

| 命令 | 用途 | 使用時機 |
|-----|------|---------|
| `/setup` | 初始化專案，讀取架構規範 | 每次啟動新對話時 |
| `/structure` | 顯示目錄架構 | 需要查看檔案組織結構時 |
| `/save-result` | 儲存分析結果指引 | 生成新的LDA結果或圖表時 |
| `/new-analysis` | 開始新分析指引 | 要進行新的LDA分析時 |
| `/write-paper` | 撰寫論文指引 | 要撰寫或修改期刊論文時 |
| `/compare` | 台美比較分析指引 | 要進行跨國比較時 |

---

## 使用方式

### 基本用法
直接在對話中輸入斜槓命令：
```
/setup
```

Claude Code 會自動展開命令內容並遵循相關指引。

---

## 使用情境範例

### 情境 1：開始新的對話
```
用戶：/setup
Claude：[讀取 CLAUDE.md 和 README.md，理解專案架構]
用戶：請幫我生成台灣LDA的主題分布圖
Claude：[依照規範儲存到 results/taiwan_lda_k7/visualizations/]
```

### 情境 2：查看專案結構
```
用戶：/structure
Claude：[顯示完整目錄架構和檔案統計]
```

### 情境 3：進行新的LDA分析
```
用戶：/new-analysis
Claude：[詢問分析國家、類型、主題數量等]
用戶：我要分析美國資料，使用K=8來看看是否有更細緻的主題
Claude：[執行分析並儲存到 results/usa_lda_k8/]
```

### 情境 4：撰寫論文
```
用戶：/write-paper
Claude：[了解現有論文和撰寫規範]
用戶：請幫我撰寫台美比較的第二章文獻回顧
Claude：[撰寫並儲存到 manuscripts/Chapter2_Literature_Review.md]
```

### 情境 5：生成比較分析
```
用戶：/compare
Claude：[了解比較分析的重點和輸出位置]
用戶：請生成台美主題對應的熱力圖
Claude：[生成並儲存到 results/comparison/Taiwan_USA_Topic_Alignment_Heatmap.png]
```

### 情境 6：儲存新結果
```
用戶：/save-result
Claude：[了解不同類型結果的儲存位置]
用戶：我剛生成了台灣的詞雲圖
Claude：請提供檔案，我會儲存到 results/taiwan_lda_k7/visualizations/
```

## 命令組合使用

### 推薦工作流程

#### 工作流程 1：新對話 → 分析 → 儲存
```
/setup              # 初始化
/new-analysis       # 開始分析
/save-result        # 儲存結果
```

#### 工作流程 2：新對話 → 比較 → 撰寫論文
```
/setup              # 初始化
/compare            # 進行比較分析
/write-paper        # 將結果寫入論文
```

#### 工作流程 3：查看結構 → 特定任務
```
/structure          # 了解現有檔案組織
/[specific-task]    # 執行特定任務
```

## 進階技巧

### 技巧 1：在命令後追加具體需求
```
/new-analysis
我要對美國資料進行K=5到K=10的主題數量優化分析
```

### 技巧 2：組合多個命令的概念
```
我需要進行台美比較分析，然後將結果寫入論文的第四章
（相當於依序使用 /compare 和 /write-paper）
```

### 技巧 3：參考命令但自行描述
```
請按照 /save-result 的規範，將我新生成的LDA模型儲存到正確位置
```

## 命令詳細說明

### `/setup` - 初始化專案
**作用**：
- 引導 Claude 讀取 CLAUDE.md 和 README.md
- 了解專案目標、現況和目錄規範
- 為後續任務建立正確的上下文

**建議使用時機**：
- 每次開始新的 Claude Code 對話
- 切換到不同的研究任務時
- 需要確保 Claude 理解專案架構時

---

### `/structure` - 顯示目錄架構
**作用**：
- 顯示完整的目錄樹狀結構
- 列出各目錄的檔案統計
- 提供快速命令索引

**建議使用時機**：
- 需要查看檔案組織方式時
- 忘記某類檔案應該放哪裡時
- 要向他人說明專案結構時

---

### `/save-result` - 儲存分析結果
**作用**：
- 提供台灣、美國、比較結果的儲存位置
- 給出檔案命名規範
- 提供 Python 程式碼範例

**建議使用時機**：
- 生成新的 LDA 模型時
- 建立視覺化圖表時
- 產生分析報告時

---

### `/new-analysis` - 開始新分析
**作用**：
- 確認資料來源位置
- 說明程式碼存放規範
- 提醒主題數量選擇框架
- 列出標準視覺化類型

**建議使用時機**：
- 要進行新的 LDA 訓練時
- 嘗試不同的主題數量（K值）時
- 需要優化模型參數時

---

### `/write-paper` - 撰寫論文
**作用**：
- 顯示現有論文檔案
- 說明論文存放位置和命名規範
- 提供撰寫建議（資料來源、圖表引用等）
- 強調方法論亮點

**建議使用時機**：
- 撰寫新章節時
- 修改現有論文時
- 需要引用分析結果時

---

### `/compare` - 台美比較分析
**作用**：
- 列出台灣和美國的資料來源
- 說明比較分析的重點面向
- 建議視覺化類型
- 顯示現有比較結果

**建議使用時機**：
- 要進行跨國比較時
- 需要對應兩國的主題時
- 分析文化差異時

## 常見問題

### Q1：命令沒有生效？
**A**：確保您使用的是 Claude Code CLI，且命令以 `/` 開頭。

### Q2：可以同時使用多個命令嗎？
**A**：可以在對話中依序使用，例如先 `/setup` 再 `/new-analysis`。

### Q3：可以自訂命令嗎？
**A**：可以！在 `.claude/commands/` 目錄下建立新的 `.md` 檔案，檔名即為命令名稱（不含 `/`）。

### Q4：命令內容可以修改嗎？
**A**：可以直接編輯 `.claude/commands/` 下的 `.md` 檔案來客製化命令內容。

## 自訂命令教學

想建立自己的命令？只需三步驟：

### 步驟 1：建立 Markdown 檔案
在 `.claude/commands/` 目錄下建立 `my-command.md`

### 步驟 2：撰寫命令內容
```markdown
這是我的自訂命令內容。

請執行以下任務：
1. 任務一
2. 任務二
3. 任務三
```

### 步驟 3：使用命令
在 Claude Code 中輸入：
```
/my-command
```
